# ✅ Исправление проблемы с доступом для роли "Владелец"

## 🐛 Проблема

Пользователи с ролью **"Владелец" (owner)** не видели навигацию и не имели доступа к функциям CRM системы, несмотря на то что им был назначен статус владельца.

## 🔍 Причина

В файле `/components/Layout.tsx` логика проверки ролей не учитывала роль `'owner'`:

### До исправления:

```typescript
// Строка 77 - проверка админских прав
const isAdmin = userRole === 'admin' || userRole === 'director';

// Строки 87-96 - массивы ролей для навигации
const allNavigation = [
  { name: 'Дашборд', href: '/', roles: ['admin', 'director', 'manager', 'sales'] },
  { name: 'Сделки', href: '/deals', roles: ['admin', 'director', 'manager', 'sales'] },
  // ... и т.д., везде отсутствует 'owner'
];
```

### Результат:
- `isAdmin` возвращал `false` для владельцев → нет доступа к настройкам
- `topNavigation.filter()` не находил пункты меню → пустая навигация
- Владельцы видели только белый экран без меню

## ✅ Решение

### 1. Добавлена роль 'owner' в проверку админских прав

```typescript
// Строка 77
const isAdmin = userRole === 'admin' || userRole === 'director' || userRole === 'owner';
```

### 2. Добавлена роль 'owner' во все массивы ролей навигации

```typescript
// Строки 87-96
const allNavigation = [
  { name: 'Дашборд', href: '/', roles: ['admin', 'director', 'manager', 'sales', 'owner'] },
  { name: 'Сделки', href: '/deals', roles: ['admin', 'director', 'manager', 'sales', 'owner'] },
  { name: 'Задачи', href: '/tasks', roles: ['admin', 'director', 'manager', 'sales', 'owner'] },
  { name: 'Календарь', href: '/production-calendar', roles: ['admin', 'director', 'manager', 'owner'] },
  { name: 'Клиенты', href: '/database', roles: ['admin', 'director', 'manager', 'sales', 'owner'] },
  { name: 'Отчеты', href: '/reports', roles: ['admin', 'director', 'manager', 'owner'] },
  { name: 'Склад', href: '/warehouse', roles: ['admin', 'director', 'manager', 'owner'] },
  { name: 'Сотрудники', href: '/employees', roles: ['admin', 'director', 'manager', 'owner'] },
  { name: 'Маркетинг', href: '/marketing', roles: ['admin', 'director', 'manager', 'owner'] },
];
```

## 📋 Иерархия ролей в системе

После исправления:

1. **👑 Владелец (owner)** = Полный доступ (как admin + director)
2. **🔑 Admin** = Полный доступ + настройки
3. **👔 Director** = Полный доступ + настройки
4. **⭐ Manager (Менеджер)** = Доступ ко всем рабочим функциям кроме некоторых настроек
5. **👁️ Observer (Наблюдатель)** = Только просмотр
6. **📞 Sales** = Работа с лидами и сделками
