# 📊 Продвинутая аналитика продаж - Реализация

## ✅ Что реализовано

### Backend Endpoints (`/supabase/functions/server/index.tsx`)

1. **GET `/analytics/abc-analysis`**
   - ABC-анализ клиентов по выручке
   - Категории: A (80% выручки), B (15%), C (5%)
   - Ранжирование с кумулятивными процентами

2. **GET `/analytics/rfm-segmentation`**
   - RFM сегментация клиентов
   - Recency (давность покупки в днях)
   - Frequency (количество сделок)
   - Monetary (общая выручка)
   - Сегменты: Champions, Loyal Customers, New Customers, At Risk, Lost, Big Spenders

3. **GET `/analytics/funnel-conversion`**
   - Анализ воронки продаж
   - Конверсия по статусам: open → won / lost
   - Win Rate (процент выигранных сделок)

4. **GET `/analytics/sales-forecast`**
   - Прогноз продаж на 3 месяца вперёд
   - Исторические данные за 12 месяцев
   - AI-инсайты от OpenAI GPT-4o-mini (анализ трендов)

5. **GET `/analytics/manager-performance`**
   - Эффективность менеджеров
   - ⚠️ Временно: показывает общую статистику (поле owner_id не существует)

6. **GET `/analytics/lost-deals`**
   - Анализ потерянных сделок
   - ⚠️ Временно: причины не детализированы (поле lost_reason не существует)

### Frontend (`/pages/SalesAnalytics.tsx`)

**6 вкладок аналитики:**

1. **Обзор**
   - Ключевые метрики (выручка, конверсия, прогноз, потери)
   - ABC распределение (pie chart)
   - RFM сегменты (bar chart)
   - AI-инсайты
   - Топ причин потерь

2. **ABC-анализ**
   - Таблица всех клиентов с категориями
   - Процент от общей выручки
   - Кумулятивный процент

3. **RFM-сегменты**
   - Таблица с RFM-скорами (R, F, M от 1 до 4)
   - Классификация по сегментам
   - Рекомендации по работе с каждым сегментом

4. **Воронка**
   - Визуализация статусов сделок
   - Конверсия и отток по этапам
   - Средний чек по статусам

5. **Прогноз**
   - График исторических данных
   - Прогноз на 3 месяца
   - Процент уверенности

6. **Менеджеры**
   - Рейтинг продавцов
   - Метрики: Win Rate, средний чек, выручка
   - ⚠️ Временно: общая статистика

## 🔧 Исправленные ошибки

### Проблема: Несуществующие поля в таблице `deals`

**Ошибки:**
```
column deals.stage does not exist
column deals.lost_reason does not exist
column deals.owner_id does not exist
```

**Решение:**

1. Заменили `stage` → `status` во всех запросах
2. Убрали `lost_reason` (причины потерь пока общие)
3. Убрали `owner_id` (эффективность менеджеров - общая статистика)

### Актуальная структура таблицы `deals`

```typescript
{
  id: string
  title: string
  amount: number
  status: 'open' | 'won' | 'lost'  // НЕ stage!
  company_id: string
  created_at: string
  updated_at: string
  stage_id: string  // ID этапа в воронке (для будущего)
  // lost_reason - НЕТ
  // owner_id - НЕТ
}
```

## 📝 Рекомендации для улучшения

### 1. Добавить поле `owner_id` в таблицу `deals`

**SQL Migration:**

**Польза:**
- Разделение статистики по менеджерам
- Персональные KPI
- Геймификация

### 2. Добавить поле `lost_reason` в таблицу `deals`

**SQL Migration:**
```sql
ALTER TABLE deals ADD COLUMN lost_reason TEXT;
```

**Польза:**
- Анализ причин отказов
- Выявление паттернов
- Улучшение процессов

### 3. Использовать `stage_id` для детальной воронки

**Идея:**
- Вместо простых статусов (open/won/lost)
- Использовать stage_id для отслеживания движения по воронке
- Qualification → Proposal → Negotiation → Won/Lost
