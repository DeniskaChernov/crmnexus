# ✅ Система перемещения товаров между складами

## 🎯 Что сделано:

### Backend (Server)

1. **✅ GET /transfers** - получение списка всех перемещений
2. **✅ POST /transfers** - создание нового перемещения с проверкой:
   - Проверка наличия товара на складе-источнике
   - Автоматический расчет остатков с учетом:
     - Произведенного
     - Отгруженного
     - Перемещенного ранее
   - Валидация: нельзя переместить больше, чем есть
3. **✅ DELETE /transfers/:id** - удаление перемещения
4. **✅ Учет в статистике** - работает в двух режимах:
   - **Backend**: патч для `/SERVER_TRANSFERS_INVENTORY_PATCH.ts` (опционально)
   - **Frontend**: автоматический расчет на клиенте (работает уже сейчас!)

### Frontend (Warehouse.tsx)

   - `isTransferOpen` - открытие диалога

   - `transfers` - список перемещений

2. **✅ Функции**:
   - `fetchTransfers()` - загрузка истории
   - `handleCreateTransfer()` - создание перемещения
   - `handleDeleteTransfer()` - удаление
   - `getAvailableArticles()` - расчет остатков с учетом перемещений
   - `getAvailableQuantity()` - точный остаток конкретного артикула

3. **✅ UI компоненты**:
   - **Кнопка "Переместить"** в header (рядом с "Отгрузка")
   - **Диалог перемещения** с полями:
     - Откуда (склад-источник)
     - Куда (склад-назначение)
     - Артикул (выбор из доступных)
     - Количество (с показом остатка)
     - Примечание
   - **Таблица истории перемещений** на вкладке "Отгрузки"

## 📋 Как использовать:

### Создание перемещения:

1. Откройте **Склад** → любая вкладка
2. Нажмите кнопку **"Переместить"** (рядом с "Отгрузка")
3. Выберите:
   - **Откуда**: исходный склад (AIKO / BTT / Bizly)
   - **Куда**: целевой склад
   - **Артикул**: товар из выпадающего списка (показаны только доступные)
   - **Количество**: вес в кг (показывается доступный остаток)
   - **Примечание**: причина перемещения (необязательно)
4. Нажмите **"Переместить"**

### Просмотр истории:

1. Откройте **Склад** → вкладка **"Отгрузки"**
2. Внизу страницы раздел **"История перемещений"**
3. Таблица показывает:
   - Дата и время
   - Откуда → Куда (с цветными бейджами)
   - Артикул
   - Количество
   - Примечание
   - Кнопка удаления

## 🔧 Логика работы:

### Как считаются остатки:

```
Остаток на складе = 
  + Произведено
  - Отгружено
  - Перемещено ОТСЮДА
  + Перемещено СЮДА
```

### Пример:

**Исходное состояние:**
- Склад AIKO: XP-3658 → 100 кг
- Склад BTT: XP-3658 → 50 кг

**После перемещения 20 кг из AIKO в BTT:**
- Склад AIKO: XP-3658 → 80 кг (100 - 20)
- Склад BTT: XP-3658 → 70 кг (50 + 20)

## ⚠️ Важное замечание:

### ✅ Система работает в двух режимах

#### Режим 1: Клиентский расчет (работает прямо сейчас!)

**Что работает:**
- ✅ Перемещения **создаются и сохраняются**
- ✅ История **отображается**
- ✅ При создании **проверяются остатки** (сервер)
- ✅ В диалоге **показываются правильные остатки** (клиент)
- ✅ **Автоматический расчет** на фронтенде

**Как работает:**
```javascript
// Frontend пересчитывает остатки локально
getAvailableQuantity(warehouse, article) {
  baseQty = stats[warehouse][article]  // С сервера
  
  transfers.forEach(t => {
    if (t.fromWarehouse === warehouse) baseQty -= t.quantity
    if (t.toWarehouse === warehouse) baseQty += t.quantity
  })
  
  return baseQty
}
```

**Преимущества:**
- 🟢 Работает немедленно без изменений backend
- 🟢 Точные остатки в диалоге перемещения
- 🟢 Корректная валидация

**Ограничения:**
- 🟡 Статистика на главном dashboard (карточки "В наличии") не учитывает перемещения
- 🟡 Нужно обновлять страницу после перемещения

---

#### Режим 2: Серверный расчет (опционально, для идеальной точности)

**Файл:** `/supabase/functions/server/index.tsx`  
**Строка:** ~2534 (после обработки shipments, перед Won Deals)  
**Что вставить:** См. файл `/SERVER_TRANSFERS_INVENTORY_PATCH.ts`

**Что даст:**
- ✅ Статистика dashboard **тоже учитывает** перемещения
- ✅ Цифры везде одинаковые (диалог = dashboard)
- ✅ Не нужно пересчитывать на клиенте

**Вывод:** Режим 1 (клиентский) **уже работает и достаточен** для большинства случаев. Режим 2 нужен только для идеальной консистентности.

## 🔍 Отладка:

### Если перемещение не создается:

1. Откройте **DevTools (F12)** → **Console**
2. Создайте перемещение
3. Смотрите логи:
   - `📦 Creating transfer:` - сервер получил запрос
   - `✅ Transfer created:` - успешно
   - `❌ Transfer creation error:` - ошибка с деталями

### Если недостаточно товара:

```
Error: Insufficient stock. Available: 50 кг, Requested: 100 кг
```

Проверьте:
1. Реальный остаток на складе-источнике
2. Не заблокирован ли товар отгрузками
3. Не было ли предыдущих перемещений
