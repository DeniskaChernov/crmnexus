# ✅ Быстрая проверка системы перемещений

## 🎯 5-минутный тест

Выполните эти шаги чтобы убедиться что всё работает:

---

## Шаг 1: Проверка начального состояния

1. Откройте **Склад** → вкладка **AIKO**
2. Найдите любой товар с остатком > 0
3. Запомните:
   - **Артикул:** (например, XP-3658)
   - **Остаток:** (например, 300 кг)

---

## Шаг 2: Первое перемещение

1. Нажмите кнопку **"Переместить"** (в header, рядом с "Отгрузка")
2. Заполните форму:
   - **Откуда:** AIKO
   - **Куда:** BTT
   - **Артикул:** выберите товар из шага 1
   - **Количество:** 50

**✅ Проверьте:**
- Под артикулом должно написать: "Доступно на складе AIKO: XXX кг"
- Цифра должна совпадать с остатком из шага 1

3. Нажмите **"Переместить"**

**✅ Ожидаемый результат:**
- Появится toast: "Перемещение создано"
- Диалог закроется

---

## Шаг 3: Второе перемещение (критический тест!)

1. **Сразу же** нажмите **"Переместить"** снова
2. Заполните:
   - **Откуда:** AIKO
   - **Куда:** Bizly
   - **Артикул:** тот же товар
   - **Количество:** 100

**✅ Критическая проверка:**
- Должно написать: "Доступно на складе AIKO: XXX кг"
- Где XXX = (начальный остаток - 50)
- Например: было 300, должно показать 250

**❌ Если показывает "0 кг" или меньше чем ожидается:**
- Значит исправление НЕ применилось
- Проверьте что файл `/components/Warehouse.tsx` обновлен

3. Нажмите **"Переместить"**

**✅ Ожидаемый результат:**
- Если остаток достаточный → Success
- Если попробовать больше чем есть → ошибка с правильной цифрой

---

## Шаг 4: Проверка истории

1. Откройте вкладку **"Отгрузки"**
2. Прокрутите вниз до раздела **"История перемещений"**
3. Должны быть видны ваши 2 перемещения:
   - AIKO → BTT: 50 кг
   - AIKO → Bizly: 100 кг

**✅ Проверьте:**
- Дата и время
- Склады (красный → зеленый)
- Артикул
- Количество

---

## Шаг 5: Проверка консоли (опционально)

1. Откройте **Developer Tools** (F12)
2. Перейдите на вкладку **Console**
3. Создайте еще одно перемещение
4. Найдите в логах:

```
📦 Creating transfer: {...}
📊 Stock calculation for XP-3658 in AIKO : {
  produced: 300,
  shipped: 0,
  transferredOut: 150,   ← Должно учитывать предыдущие!
  transferredIn: 0,
  availableStock: 150,
  requestedQuantity: ...
}
✅ Transfer created: {...}
```

**✅ Проверьте:**
- `transferredOut` должен быть = сумме всех предыдущих перемещений
- `availableStock` должен быть правильным

---

## 🎯 Критерии успеха:

### ✅ Всё работает если:

- [x] В диалоге показывается правильный остаток
- [x] Остаток **уменьшается** после каждого перемещения
- [x] Можно сделать **несколько** перемещений подряд
- [x] **Нет ошибки** "Available: 0 кг"
- [x] При попытке переместить **больше чем есть** → понятная ошибка
- [x] История сохраняется и отображается

### ❌ Есть проблема если:

- [ ] Показывает "Available: 0 кг" после первого перемещения
- [ ] Остаток не изменяется
- [ ] Нельзя сделать второе перемещение
- [ ] Ошибка без деталей

---

## 🐛 Если что-то не работает:

### Проблема: "Available: 0 кг"

**Решение:**
1. Убедитесь что файл `/components/Warehouse.tsx` обновлен
2. Проверьте функции `getAvailableArticles()` и `getAvailableQuantity()`
3. Перезагрузите страницу (Ctrl+R)

### Проблема: Серверная ошибка

**Решение:**
1. Откройте консоль браузера (F12)
2. Посмотрите на Network → последний запрос к `/transfers`
3. Проверьте Response
4. Убедитесь что файл `/supabase/functions/server/index.tsx` обновлен

### Проблема: Нет логов в консоли

**Решение:**
1. Это backend логи, они в серверной консоли
2. Frontend логи смотрите в браузере
3. Если нужны серверные логи → проверьте Supabase logs
