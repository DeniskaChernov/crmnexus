#### 8. **GET `/analytics/sales-velocity`**
**Описание:** Скорость продаж - как быстро сделки движутся по воронке

**Query params:** `startDate`, `endDate`

**Response:**
```json
{
  "avgSalesCycle": 25,
  "avgWonCycle": 22,
  "totalDeals": 200,
  "wonDeals": 140,
  "winRate": 70.0,
  "avgDealValue": 500000,
  "velocity": 15909090,
  "cycleDaysDistribution": {
    "0-7": 30,
    "8-30": 100,
    "31-60": 50,
    "61+": 20
  }
}
```

**Формула Velocity:**
```
Velocity = (Количество сделок × Средний чек × Win Rate) / Длительность цикла
```

**Как увеличить Velocity:**
- ↑ Увеличить количество сделок в воронке
- ↑ Повысить средний чек
- ↑ Улучшить win rate
- ↓ Сократить длительность цикла сделки

## 📊 Ключевые метрики и KPI

### 1. Retention Rate (Удержание)
**Формула:** `Повторные клиенты / Все клиенты когорты × 100%`

**Норма:** 
- 🟢 > 50% - отличное удержание
- 🟡 30-50% - среднее
- 🔴 < 30% - проблемы с лояльностью

### 2. LTV (Lifetime Value)
**Формула:** `Общая выручка когорты / Количество клиентов`

**Применение:** Понимание ценности клиента для планирования CAC

### 3. Sales Velocity
**Формула:** `(Сделок × Средний чек × Win Rate) / Длительность цикла`

**Единица:** сум/день

**Применение:** Комплексная оценка эффективности продаж

### 4. Win Rate
**Формула:** `Выигранные сделки / Закрытые сделки × 100%`

**Норма:**
- 🟢 > 50% - отличная конверсия
- 🟡 30-50% - средняя
- 🔴 < 30% - нужна оптимизация

### 5. Average Sales Cycle
**Единица:** дни

**Норма (зависит от отрасли):**
- B2C: 1-7 дней
- B2B малый бизнес: 7-30 дней
- B2B средний: 30-60 дней
- B2B крупный: 60+ дней

## 🎯 Бизнес-сценарии использования

### Сценарий 1: Увеличение выручки
**Задача:** На 20% увеличить выручку в следующем квартале

**Анализ:**
1. **ABC-анализ** → выявить топ-20% клиентов (категория A)
2. **RFM** → найти сегмент "At Risk" среди категории A
3. **Когорты** → определить когорты с низким retention
4. **Velocity** → понять, какой фактор замедляет продажи

**Действия:**
- Персональное внимание клиентам категории A
- Реактивация "At Risk" клиентов
- Программа лояльности для когорт с низким retention
- Оптимизация узких мест в воронке

### Сценарий 2: Оптимизация работы менеджеров
**Задача:** Повысить эффективность отдела продаж

**Анализ:**
1. **Velocity** → измерить текущую скорость
2. **Sales Cycle** → найти долгие сделки
3. **Heatmap** → определить лучшее время для контактов
4. **Funnel** → найти этапы с максимальным оттоком

**Действия:**
- Обучение работе с возражениями на проблемных этапах
- Планирование звонков в оптимальное время
- Автоматизация рутинных процессов
- KPI по сокращению цикла сделки

### Сценарий 3: Планирование бюджета
**Задача:** Составить прогноз на следующий квартал

**Анализ:**
1. **Прогноз AI** → 3 метода + доверительный интервал
2. **Trend** → определить направление (рост/снижение)
3. **Cohorts LTV** → понять ценность новых клиентов
4. **Historical data** → проверить сезонность

**Действия:**
- Планирование бюджета на основе прогноза
- Учет доверительного интервала для рисков
- Инвестиции в привлечение с учетом LTV
- Корректировка целей под тренд

## 🎓 Глоссарий терминов

- **ABC-категория** - классификация клиентов по вкладу в выручку (A=80%, B=15%, C=5%)
- **RFM Score** - комбинация 3 показателей: Recency (давность), Frequency (частота), Monetary (деньги)
- **Когорта** - группа клиентов с общей характеристикой (месяц первой покупки)
- **LTV** - Lifetime Value, ожидаемая выручка за все время работы с клиентом
- **Retention** - удержание, % клиентов, вернувшихся за повторной покупкой
- **Win Rate** - % выигранных сделок от закрытых
- **Sales Cycle** - среднее время от создания до закрытия сделки
- **Velocity** - скорость генерации выручки (сум/день)
- **Доверительный интервал** - диапазон значений прогноза с заданной вероятностью (95%)
