# 🔧 Настройка Telegram Webhook для BTT NEXUS CRM

## Проблема

Telegram не может отправлять webhook запросы на Supabase Edge Function из-за JWT-авторизации. Telegram не поддерживает отправку кастомных заголовков `Authorization`, поэтому все запросы получают ошибку **401 Unauthorized**.

## Решение

Необходимо **отключить JWT verification** для Edge Function `server`, чтобы разрешить публичный доступ к эндпоинту `/make-server-f9553289/telegram-webhook`.

---

## 📋 Инструкция по настройке

### Вариант 1: Через Supabase Dashboard (Рекомендуется)

1. **Откройте Supabase Dashboard**
   - Перейдите на [supabase.com/dashboard](https://supabase.com/dashboard)
   - Выберите ваш проект BTT NEXUS

2. **Перейдите в Edge Functions**
   - В левом меню нажмите **Edge Functions**
   - Найдите и откройте функцию **`server`**

3. **Отключите JWT Verification**
   - В настройках функции найдите переключатель **"Enforce JWT Verification"**
   - **Отключите** его (toggle OFF)
   - Нажмите **Save** / **Сохранить**

4. **Обновите Webhook URL**
   - Вернитесь в CRM → **Настройки** → **Интеграции** → **Отладка**
   - Нажмите кнопку **"Исправить Webhook"**
   - Убедитесь, что статус изменился на "OK"

5. **Проверьте работу**
   - Отправьте боту команду `/ping`
   - Должен прийти ответ "🏓 Pong! Система работает."
   - Попробуйте отправить рецепт или отчет производства

---

### Вариант 2: Через Supabase CLI

Если у вас установлен Supabase CLI, выполните:

```bash
supabase functions deploy server --no-verify-jwt
```

Эта команда задеплоит функцию с отключенной проверкой JWT.

---

### Вариант 3: Через config.toml (для локальной разработки)

Файл `/supabase/config.toml` уже создан с необходимыми настройками:

```toml
[functions.server]
verify_jwt = false
```

При следующем деплое через CLI эта настройка будет применена автоматически.

---

## ✅ Проверка работы

После настройки:

1. **Webhook URL должен быть:**
   ```
   https://ваш-проект.supabase.co/functions/v1/make-server-f9553289/telegram-webhook
   ```

2. **В очереди Telegram должно быть 0 сообщений**

3. **Отправьте боту `/ping`** - должен ответить

4. **Проверьте логи** в разделе "История запросов"

---

## 🔒 Безопасность

**Важно:** Отключение JWT verification делает эндпоинт публичным. 

**Рекомендации:**
- Эндпоинт `/make-server-f9553289/telegram-webhook` обрабатывает только запросы от Telegram
- Все остальные эндпоинты защищены авторизацией
- В будущем можно добавить проверку секретного токена Telegram для дополнительной безопасности

---

## 📞 Что делать, если не работает

1. **Проверьте статус webhook** в разделе "Отладка"
2. **Убедитесь, что JWT verification отключен** в настройках функции
3. **Нажмите "Исправить Webhook"** после изменения настроек
4. **Проверьте логи** - должны появиться записи о запросах
5. **Отправьте `/ping`** для проверки связи

Если проблема сохраняется - проверьте консоль браузера и логи Supabase Dashboard → Edge Functions → server → Logs.
